# RAG 技术简介

## 什么是 RAG

RAG（Retrieval-Augmented Generation，检索增强生成）是一种将信息检索技术与大型语言模型（LLM）相结合的架构。它通过在生成回答之前先从外部知识库中检索相关信息，从而提升回答的准确性和时效性。

## RAG 的核心组件

### 1. 检索器（Retriever）
检索器负责从知识库中找到与用户查询相关的文档。常用的技术包括：
- 向量检索：使用 Embedding 模型将文本转换为向量，通过相似度计算找到相关文档
- 关键词检索：传统的 BM25、TF-IDF 等方法
- 混合检索：结合向量和关键词的优势

### 2. 生成器（Generator）
生成器通常是一个大型语言模型，它接收检索到的上下文和用户查询，生成最终的回答。

### 3. 知识库（Knowledge Base）
存储文档和向量索引的数据库，常用的有：
- 向量数据库：Milvus、Pinecone、Weaviate、Qdrant
- 传统数据库 + 向量插件：PostgreSQL with pgvector

## RAG 的工作流程

1. **索引阶段**：
   - 文档加载和解析
   - 文本分块
   - 向量化
   - 存储到向量数据库

2. **查询阶段**：
   - 用户输入查询
   - 查询向量化
   - 向量相似度检索
   - 上下文组装
   - LLM 生成回答

## RAG 的优势

- **减少幻觉**：基于检索到的事实生成回答，减少模型编造信息
- **知识更新**：可以动态更新知识库，无需重新训练模型
- **可解释性**：可以展示检索到的来源文档
- **成本效益**：比微调模型更经济

## RAG 的挑战

- 检索质量直接影响生成质量
- 大文档的分块策略需要精心设计
- 多轮对话中的上下文管理
- 检索和生成的延迟优化

## 应用场景

- 企业知识库问答
- 智能客服
- 法律文档分析
- 医疗知识查询
- 代码文档助手
