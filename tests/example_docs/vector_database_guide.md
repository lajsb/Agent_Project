# 向量数据库指南

## 什么是向量数据库

向量数据库是一种专门用于存储和检索高维向量的数据库系统。它通过近似最近邻（ANN）算法，快速找到与查询向量相似的向量。

## 核心概念

### 向量嵌入（Embedding）
将文本、图像等非结构化数据转换为数值向量的过程。这些向量在高维空间中的距离反映了原始数据的语义相似度。

### 相似度度量

1. **余弦相似度（Cosine Similarity）**
   - 计算两个向量之间的夹角
   - 范围：[-1, 1]，通常用于归一化后的向量
   - 适合关注方向而非绝对大小的场景

2. **欧氏距离（Euclidean Distance / L2）**
   - 计算向量间的直线距离
   - 范围：[0, ∞)
   - 适合关注绝对差异的场景

3. **内积（Dot Product / IP）**
   - 向量的点积
   - 范围：(-∞, ∞)
   - 当向量已归一化时，等同于余弦相似度

## 流行的向量数据库

### Milvus
- 开源、云原生
- 支持十亿级向量
- 分布式架构
- 丰富的索引类型（IVF、HNSW 等）

### Pinecone
- 全托管云服务
- 简单易用
- 自动扩缩容
- 按用量付费

### Weaviate
- 开源 + 云服务
- GraphQL 接口
- 模块化架构
- 支持混合搜索

### Qdrant
- 开源，Rust 编写
- 高性能
- 过滤器支持好
- 易于部署

## 索引算法

### IVF（Inverted File Index）
- 将向量空间划分为多个区域
- 先找到最近区域，再精确搜索
- 平衡速度和精度的参数：nlist、nprobe

### HNSW（Hierarchical Navigable Small World）
- 基于图的索引
- 多层图结构加速搜索
- 参数：M（每个节点的连接数）、efConstruction、efSearch

### FLAT
- 暴力搜索，精确但慢
- 适合小数据集

## 最佳实践

1. **选择合适的维度**：OpenAI 的 text-embedding-3-small 是 1536 维
2. **索引选择**：
   - 数据集 < 100万：HNSW
   - 数据集 > 100万：IVF_FLAT 或 IVF_SQ8
3. **定期优化**：对于动态数据，定期重建索引
4. **监控性能**：跟踪查询延迟和召回率

## 在 RAG 中的应用

向量数据库是 RAG 系统的核心组件：

1. **文档向量化**：将文档分块后转换为向量
2. **索引构建**：创建高效的向量索引
3. **相似度搜索**：根据查询向量找到相关文档
4. **结果返回**：将检索到的内容作为上下文提供给 LLM

选择合适的向量数据库对于 RAG 系统的性能至关重要。
